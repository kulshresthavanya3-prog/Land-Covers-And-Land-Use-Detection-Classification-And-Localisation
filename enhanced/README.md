## 📌 Overview

This dataset contains the **enhanced version** of all 219 raw LISS-III/IV satellite scenes for the Shillong region — produced by a three-stage pipeline of **valid-pixel masking → denoising → CLAHE contrast enhancement**. These are the images actually sent for QGIS/VIA annotation and used as the source for 256×256 patch extraction.

For the raw source scenes, see the [training dataset README](./README_training_dataset.md). For the annotation color legend, see the [VIA annotation README](./README_via_annotations.md).

---

## 🧼 Enhancement Pipeline

Each raw scene passes through three deterministic stages before being marked "enhanced":

### 1️⃣ Valid-Pixel Masking
A binary mask is generated by thresholding the grayscale image at **intensity 8**. Pixels at or below this threshold are treated as **sensor fill / border artefacts** and excluded from all downstream enhancement so black borders never get artificially brightened.

### 2️⃣ Denoising
`cv2.fastNlMeansDenoisingColored` suppresses sensor noise while preserving edge detail:

| Parameter | Value |
|---|---|
| `h` | 5 |
| `hColor` | 5 |
| `templateWindowSize` | 7 |
| `searchWindowSize` | 21 |

### 3️⃣ CLAHE Contrast Enhancement
**Contrast-Limited Adaptive Histogram Equalisation** is applied on the **L-channel in LAB color space**, then masked so black-border pixels remain zero:

| Parameter | Value |
|---|---|
| `clipLimit` | 2.0 |
| `tileGridSize` | 8 × 8 |
| Channel | L (lightness), LAB colorspace |

```
Raw Scene → Valid-Pixel Mask → Denoise (fastNlMeans) → CLAHE (LAB L-channel) → Enhanced Scene
```

## ⚠️ Known Limitations

- **Fixed threshold (intensity 8)** for valid-pixel masking is not scene-adaptive — very dark but genuinely valid terrain (e.g., deep shadow on steep slopes) could be misclassified as border artefact on a minority of scenes.
- **CLAHE parameters are global** (`clipLimit=2.0`, `tileGrid=8×8` for all 219 scenes) — no per-scene tuning was applied, so enhancement quality may vary slightly with each scene's original dynamic range.
- **No before/after quantitative image-quality metrics** (e.g., SSIM, contrast ratio) are currently published — enhancement quality has been assessed visually, not numerically.
- Enhanced scenes with <70% valid pixels are still included in `outputs/enhanced/`, but their derived 256×256 tiles are filtered out later at the patch-extraction stage, not here.
